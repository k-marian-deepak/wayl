# GUIDELINES FOR CONTRIBUTION

Please follow the guidelines in this file while contributing to accept your contribution.

## Step up for contribution
- Fork the repo
- Clone the repo
- Install the requirements (refer to [README.md](../README.md#running-the-application-step-by-step))

## Branch & Pull Request Rules
While contributing make sure to do only one edit i.e. respective edit of one issue don't do multiple edits for a single PR.

### Naming of the branch
* `fix-*` - `fix-` prefix for code changes fixing issues (e.g., `fix-issue-1876`).
* `doc-*` - `doc-` prefix for documentation changes (e.g., `doc-code-of-conduct`).
